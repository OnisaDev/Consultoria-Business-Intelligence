## 📋 Descripción

Caso práctico de consultoría integral para el **diseño de una arquitectura de Business Intelligence** para una red nacional de academias de inglés. El proyecto abarca desde el análisis inicial de necesidades hasta el diseño del Data Warehouse, el proceso ETL y los dashboards por rol.

Trabajo académico del módulo de **Big Data (HLC)** del CFGS DAM.

---

## 🎯 Objetivo

Diseñar una solución BI personalizada que permita a la empresa:

- Centralizar datos dispersos entre Excel, CRM, facturación y formularios web
- Automatizar informes generados manualmente
- Comparar rendimiento entre sedes
- Detectar tendencias de matriculación y abandono
- Controlar el ROI de campañas de marketing

---

## 📐 Fases del proceso de consultoría

| Fase | Descripción |
|---|---|
| 1. Kick-off | Reunión inicial, toma de requisitos, identificación de perfiles de usuario |
| 2. Análisis de fuentes | Inventario de datos: Excel, CRM, facturación, formularios, BBDD |
| 3. Calidad del dato | Normalización, limpieza, homogeneización entre sedes |
| 4. Definición de KPIs | Matriculaciones, asistencia, beneficio, CAC, ROI de campañas |
| 5. Arquitectura BI | Diseño ETL, Data Warehouse (modelo en estrella), seguridad y roles |
| 6. Desarrollo DW + ETL | Construcción técnica, scripts de carga, pruebas de integridad |
| 7. Dashboards | Power BI: vistas por CEO, directores, profesores, financiero y marketing |
| 8. Despliegue | Formación, publicación, backups, mantenimiento y escalabilidad |
